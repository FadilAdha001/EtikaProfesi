Pertemuan 11

Modul ini berfokus pada penerapan Hak Kekayaan Intelektual (HAKI) di produk dan layanan digital, membahas perbedaan hak cipta, paten, merek dagang, serta etika penggunaan lisensi open-source.

1. Capaian Pembelajaran

  • Menjelaskan konsep HAKI: Hak Cipta, Paten, dan Merek Dagang.
  • Membedakan lisensi perangkat lunak (MIT, GPL, Apache).
  • Menganalisis studi kasus dan menyusun rekomendasi mitigasi etis.

2. Ringkasan Materi (Konsep Kunci)

Hak Cipta (Copyright)
Melindungi ekspresi ide seperti source code, dokumentasi, desain UI. Perlindungan otomatis saat karya diciptakan.
Hak Paten (Patent)
Melindungi penemuan teknis atau metode baru yang bersifat fungsional; memerlukan pendaftaran.
Merek Dagang (Trademark)
Melindungi nama, logo, dan identitas visual untuk mencegah kebingungan konsumen.
Lisensi Open Source — ringkasan
MIT: sangat permisif, boleh digunakan dan dimodifikasi selama ada atribusi.
GPL: copyleft — turunan harus berlisensi sama saat didistribusikan.
Apache: permisif + klausul paten.

3. Studi Kasus Interaktif — "KopiTracker"

Skenario singkat: Seorang developer merilis aplikasi KopiTracker yang membantu pengguna menemukan kedai kopi lokal.

  • Kode peta: library open-source (lisensi MIT).
  • Logo & palet warna mirip brand kopi besar.
  • Nama aplikasi mengandung kata yang mirip merek terkenal.

4. Tugas & Kuis

Tugas Individu

Buat analisis 250–350 kata mengenai studi kasus di atas. Sertakan: jenis HAKI, risiko hukum & etika, serta rekomendasi mitigasi profesional.

Pengumpulan: Google Form / Google Classroom (sertakan link ke repo GitHub atau PDF laporan).

Kuis Singkat (5 soal)

1. Lisensi MIT mengizinkan?
2. Paten biasanya melindungi?
3. Trademark melindungi?
4. Contoh lisensi copyleft adalah?
5. Langkah pertama bila logo mirip brand besar?

5. Sumber & Bacaan Tambahan

  • Ringkasan Undang-Undang HAKI (bisa diunggah ke Drive dan dicantumkan link).
  • Dokumentasi lisensi open-source: MIT, GPL, Apache.
  • Contoh README & LICENSE untuk publikasi di GitHub.

Catatan: kontak dosen / asisten dapat ditambahkan di bagian kanan (sidebar).

Ringkasan Cepat

  1. Periksa lisensi dependency sebelum rilis.
  2. Cantumkan file LICENSE & atribusi di About.
  3. Pastikan nama/logo unik untuk menghindari pelanggaran trademark.
